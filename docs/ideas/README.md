# Ideas - 機能アイデア管理

このディレクトリは、実装予定の機能について、Claude との壁打ちで得た「要件・設計・懸念事項」をマークダウンで保存するための場所です。

## ディレクトリ構造

```
ideas/
├── README.md              # このファイル
├── pending/               # 🟡 まだ実装していないアイデア
│   ├── 20250412-ユーザープロフィール編集-initial-requirements.md
│   └── 20250415-支出予測機能-initial-requirements.md
└── done/                  # ✅ すでに実装済みのアイデア
    ├── 20250401-ホーム画面改善-initial-requirements.md
    └── 20250405-カテゴリ管理機能-initial-requirements.md
```

## ファイル命名ルール

**ファイル名フォーマット**: `[YYYYMMDD]-[機能名]-initial-requirements.md`

**例**:
- `20250412-ユーザープロフィール編集-initial-requirements.md`
- `20250415-月別支出予測機能-initial-requirements.md`
- `20250420-レポート画面改善-initial-requirements.md`

**重要**: `[機能名]` には、後で `/add-feature` で使用する機能名と同じものを使用してください。

## フロー

### 新機能のアイデア → 実装完了までのフロー

```
1. このチャット画面で壁打ち
   ↓
2. 合意内容を ideas/pending/ に保存
   └─ 例: ideas/pending/20250412-ユーザープロフィール編集-initial-requirements.md
   ↓
3. /add-feature で実装開始
   └─ `/add-feature ユーザープロフィール編集`
   ↓
4. 実装完了 ✅
   ↓
5. ideas/pending/ から ideas/done/ に移動
   └─ mv ideas/pending/[ファイル] ideas/done/[ファイル]
```

## ファイル内容のテンプレート

各アイデアファイルには、以下の内容を含めることをお勧めします：

```markdown
# [機能名] - 初期要件

## 概要

**一言説明**:
[機能の簡潔な説明]

**目的**:
[なぜこの機能が必要か]

## 要件（やること）

- [ ] [要件1]
- [ ] [要件2]
- [ ] [要件3]

## 非要件（やらないこと）

- [スコープ外のこと1]
- [スコープ外のこと2]

## 設計方針（どうやるか）

### UI/UX

[ユーザーフロー、画面配置などの説明]

### 技術的な方針

[フロントエンド実装方針、データモデルなど]

## 懸念事項・検討点

- [懸念1]: [詳細]
- [懸念2]: [詳細]

## 実装予定時期

[予定またはいつでも可能か]

## 壁打ちの記録

**最終更新**: [日付]
**合意内容**: [壁打ちで決定した最終的な仕様]
```

## pending から done への移動タイミング

以下のいずれかのタイミングで移動します：

1. **実装完了 & マージされたとき**
   ```bash
   mv ideas/pending/[ファイル] ideas/done/[ファイル]
   git add .
   git commit -m "Move [機能名] from pending to done"
   ```

2. **または tasklist.md に振り返りを記録したとき**
   ```bash
   # .steering/[日付]-[機能名]/tasklist.md に振り返り完了 → ファイル移動
   ```

## 使用例

### 例 1: ユーザープロフィール編集機能

**壁打ち記録**:
```markdown
# ユーザープロフィール編集 - 初期要件

## 概要

**一言説明**: ユーザーが自身のプロフィール情報を編集できる機能

**目的**: ユーザーが名前、メールアドレス、プロフィール画像を更新可能にする

## 要件

- [ ] プロフィール画面を作成
- [ ] 名前、メールアドレスの編集
- [ ] プロフィール画像のアップロード
- [ ] 変更を保存
- [ ] バリデーション実装

## 設計方針

### UI/UX
- 設定ページに新しいセクション「プロフィール編集」を追加
- 画像プレビュー表示
- 保存ボタンのみで即座に保存

### 技術的方針
- React Hook Form + Zod でバリデーション
- Axios で API 呼び出し
- FormData で画像アップロード

## 懸念事項

- 画像ファイルサイズの制限（5MB？）
- メールアドレス変更時の再認証が必要か？
```

**その後**:
```bash
# 実装完了後
mv ideas/pending/20250412-ユーザープロフィール編集-initial-requirements.md \
   ideas/done/20250412-ユーザープロフィール編集-initial-requirements.md
```

## チェックリスト（実装時）

実装前に以下を確認：

- [ ] `ideas/pending/` に要件ファイルが存在するか
- [ ] ファイル名が `/add-feature` で使用する機能名と一致しているか
- [ ] `/add-feature [機能名]` を実行
- [ ] 実装完了後、ファイルを `ideas/done/` に移動したか

## Git での管理

### staging の例

```bash
# 新しいアイデアを pending に追加
git add ideas/pending/20250412-ユーザープロフィール編集-initial-requirements.md
git commit -m "Add idea: ユーザープロフィール編集"

# 実装完了後、done に移動
git add ideas/pending/ ideas/done/
git commit -m "Move ユーザープロフィール編集 to done"
```

## メリット

✅ **アイデア履歴が残る**：どんな機能を追加したのか、なぜ追加したのか後から参照可能
✅ **pending/done で進捗が一目瞭然**：どの機能がまだ実装待ちか、どの機能が完了したか
✅ **ドキュメント化の習慣**：口頭だけでなく、必ずマークダウンに記録される
✅ **知識共有**：他の人が壁打ち内容を参照できる
✅ **リード**：「あの機能、どういう理由で実装したんだっけ」という質問に対応可能
