# 固定費機能 - 初期要件

## 概要

**一言説明**: 毎月同じ日に自動で複製される支出（家賃、保険、定期購読など）を登録・管理する機能。

**目的**: ユーザーが毎月の固定費を手動で入力する手間を削減し、家計管理の負担を軽減する。

## 要件（やること）

- 固定費テーブルの作成（マイグレーション）
- 固定費 CRUD API の実装
- 毎月自動複製バッチ処理の実装（毎月1日 05:00 JST）
- 支出作成フォームに「固定費として登録」チェックボックスを追加
- 固定費管理画面（一覧・追加・編集・削除）を実装
- カレンダービューで固定費バッジを表示
- 固定費の金額変更時：「この日以降に適用」「すべて変更」オプション

## 非要件（やらないこと）

- 複製の一括削除機能（「このシリーズ全部削除」など）
- 複製された支出の個別編集（複製後は通常の支出として編集可能）
- 固定費の有効/無効切り替え（削除のみ）

## 設計方針（どうやるか）

### データベース設計

- **新規テーブル**: `fixed_expenses` - 固定費の定義を管理
- **カラム追加**: `contents` に `is_fixed_expense`, `fixed_expense_day`, `fixed_expense_id` を追加
- **オプション**: `fixed_expense_history` テーブルで金額変更履歴を管理

### API 設計

```
GET  /api/fixed-expenses          - 固定費一覧取得
POST /api/fixed-expenses          - 固定費作成
PUT  /api/fixed-expenses/{id}     - 固定費更新
DELETE /api/fixed-expenses/{id}   - 固定費削除
POST /api/contents (with is_fixed_expense flag)  - 固定費フラグ付き支出作成
```

### バックエンド実装方針

- **Laravel Scheduler** で毎月1日 05:00 JST に自動実行
- **チャンク処理**（100件単位）で DB 負荷を軽減
- **リトライ機構**: 失敗時は同日 06:00 に再実行
- **エラー通知**: 管理者へメール/Slack 通知

### フロントエンド実装方針

- **固定費管理ページ**: 一覧・追加・編集・削除画面
- **支出フォーム修正**: チェックボックス追加
- **カレンダービュー**: 「固定費」バッジ表示

## 懸念事項・検討点

- **毎月の何日実行？**: 1～31日選択可能。29～31日は月末に自動調整。
- **自動複製のタイミング**: 毎月1日朝5時は日本人の最も使用しない時間帯 → 理由付き
- **金額変更時の過去データ**: 変更前の複製済み支出は変更しない（変更以降のみ新金額適用）
- **複数サーバー環境**: `.onOneServer()` で重複実行を防止
