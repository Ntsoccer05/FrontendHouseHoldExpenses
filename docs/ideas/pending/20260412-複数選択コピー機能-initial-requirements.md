# 複数選択コピー機能 - 初期要件

## 概要

**一言説明**: 同じ日付の複数トランザクションを選択して、別の日付に一括コピーできる機能。

**目的**: ユーザーが複数の支出を繰り返す場面（例：買い物リストが同じ日に複数項目がある場合）で、毎回同じ項目を入力する手間を削減する。

## 要件（やること）

- トランザクション一覧で複数選択モードを実装
- 各トランザクションにCheckbox を表示（選択モード時のみ）
- 「N件をコピー」ボタンで一括コピー
- コピー先の日付を選択するダイアログを表示
- 複数トランザクション一括コピー API を実装
- コピー後にデータを自動更新

## 非要件（やらないこと）

- 一括削除機能
- シリーズ編集機能
- コピー後の自動編集機能

## 設計方針（どうやるか）

### UI/UX

- **選択モード**: トランザクション一覧に「複数選択」ボタンを追加
- **Checkbox**: 選択モード時、各トランザクション左側に Checkbox を表示
- **一括操作ボタン**: 「N件をコピー」「キャンセル」ボタンを表示
- **コピーダイアログ**: コピー先の日付を MUI Dialog で選択

### 複製後の画面遷移

**基本方針**: コピー後は **デフォルトでは画面遷移しない**（ただしキャッシュはクリア）

**理由**:
- ユースケース: 複数月への複製時に、1回1回対象月に移動されると煩わしい
- 例：3月から4月・5月・6月に複製する場合、1回1回6月に移動されたくない

**実装**:
1. 複製成功後、該当月のキャッシュをクリア（`invalidateMonthCache()`）
2. 完了メッセージに **オプショナルなリンク** を表示：
   ```
   SnackBar: "5月に5件をコピーしました"
   ├─ 自動消失（3秒後）
   └─ Action ボタン: "移動して確認" → クリックで対象月に移動
   ```
3. ユーザーが **必要に応じて** 確認できる

### 技術的な方針

- **フロントエンド**: TransactionMenu.tsx に isSelectMode, selectedIds state を追加
- **選択状態UI**: Card に borderLeft: 4px solid でハイライト
- **API呼び出し**: `/api/contents/copy-multiple` に複数ID + 日付を送信
- **型修正**: Transaction.id は string なので、selectedIds: string[] に統一
- **データ更新**: 
  - 同一月内複製：`invalidateMonthCache(currentMonth)` でクリア → 自動再取得
  - 異月複製：`invalidateMonthCache(destinationMonth)` でクリア → 次回表示時に新データ取得
- **SnackBar アクション**: "移動して確認" ボタンで `setCurrentMonth(destinationDate)` 呼び出し

## 懸念事項・検討点

- 同一日付内での選択のみ対応（異なる日付のトランザクションを選択したら？）→ 初回は同一日付のみに限定
- API がコール回数制限の対象か確認が必要
- 大量選択時（100件超）のパフォーマンス検証
- **複製後の画面遷移**: 複数月への複製時に移動しないのが UX 的に最適か確認済み（必要に応じて手動で移動可）
- **キャッシュ無効化**: 同一月内複製と異月複製で処理が異なる
